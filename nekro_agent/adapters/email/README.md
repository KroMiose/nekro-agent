# Email 适配器

Email 适配器把邮箱收发、邮件同步、附件流转和邮件内容检索接入 Nekro Agent。它既可以作为新邮件入口，把邮件内容转换为独立会话中的消息，也可以作为邮件工作台，集中管理多个收发信账户、查看历史邮件、转发附件并排查收件链路问题。

当前版本：2.0.0

## 核心能力

- 多账户管理：支持为每个邮箱账户单独配置启用状态、收信开关、发信开关和默认发件人。
- 多服务商接入：内置 QQ 邮箱、163 邮箱、Gmail、Outlook 和自定义 IMAP/SMTP 配置。
- 官方 OAuth 登录：支持 Gmail 与 Outlook / Microsoft 365 的官方登录授权，减少手动维护密码或授权码的成本。
- 自动轮询收件：按配置间隔拉取收件箱邮件，可选择仅处理未读邮件并在处理后标记已读。
- 邮件会话化：同一邮箱账户会映射为独立聊天频道，新邮件正文、发件人、主题和附件信息会进入 Agent 上下文。
- 邮件列表与详情：前端提供已同步邮件列表、账户过滤、正文查看、邮件头信息和附件预览入口。
- 附件保存与转发：自动保存邮件附件，并可通过内置工具转发到其他聊天频道。
- 发信能力：可使用启用发信的账户发送文本邮件，支持默认发件人选择。
- 邮件总结与查询：内置工具可总结近期邮件、查询指定邮件完整内容和附件信息。
- 手动拉取诊断：账户页可手动拉取收件箱最近邮件，并展示文件夹选择、搜索条件、处理数量、错误阶段等调试信息。
- 运行状态通知：可配置 Bot 上下线时通过指定邮箱发送状态通知。

## 适合什么场景

- 邮箱值守：持续接收通知类邮箱、项目邮箱或业务邮箱的新邮件。
- 来信整理：快速总结近期邮件、定位指定主题或发件人的消息。
- 附件处理中转：保存邮件附件，并转发给群聊、私聊或其他适配器渠道。
- 自动化邮件助手：把邮件正文纳入 Agent 上下文，继续做提取、改写、总结和回复草稿。
- 收件问题排查：通过手动拉取诊断确认是否选中正确收件箱、是否搜到邮件、是否处理失败。

## 使用前你需要准备

- 至少一个可用邮箱账户。
- 对应邮箱已开启 IMAP/SMTP，或已创建可用于 OAuth 的 Gmail / Microsoft 应用。
- 对密码登录账户，建议使用授权码或应用专用密码。
- 对 Outlook / Microsoft 365，建议优先使用官方登录，并确认应用权限包含邮件读取/发送所需 scope。
- 如果邮箱服务商在海外或网络受限，可配置 OAuth / IMAP / SMTP 代理。

## 使用时建议注意

- 轮询间隔不要设得过短，避免触发邮箱服务商频控。
- 如果开启“读取后标记已读”，手动拉取和自动轮询都会影响邮件未读状态。
- 手动拉取默认用于诊断和同步最近收件箱邮件，可能会让已存在邮件再次进入处理流程。
- 如果配置了多个发信账户，建议明确默认发件人，避免外发账户不符合预期。
- 涉及隐私或正式外发内容时，发送前请确认收件人、主题和正文。
