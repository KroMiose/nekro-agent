# 飞书适配器

适合把 Agent 带进飞书工作区，在团队聊天里承担答疑、通知、资料发送和协作辅助工作。  
如果你的团队主要在飞书沟通，这会是让 Agent 被自然使用起来的高频入口。

## 适合什么场景

- 团队问答：回答文档、制度、流程、知识库相关问题
- 协作辅助：在讨论中补充资料、生成说明、跟进任务
- 日常通知：把系统状态、日报、提醒推送到群聊或私聊
- 轻量办公助手：让团队成员直接在聊天窗口里获得帮助

## 你可以用它做什么

- 在飞书群聊和私聊中接收并回复消息
- 支持 @提及、图片和文件，沟通体验更完整
- 可在处理过程中给出即时反馈，减少“已读未回”感
- 适合团队问答、协同办公、知识助手和消息通知场景

## 使用前你需要准备

- 一个可正常使用的飞书应用
- 机器人已经加入你的目标群组或能处理私聊消息
- 明确你想让 Agent 处理的知识、通知或协作内容

## 使用体验上有什么特点

- 飞书里的 @、图片、文件等交互体验比较完整
- 很适合把知识问答和团队协作揉到同一个入口里
- 对于“边聊边处理”的工作流，这类适配器通常更顺手

## 使用时建议注意

- 群聊场景建议先从固定主题的群开始，便于建立预期
- 如果接入知识库或资料库，建议优先测试答案稳定性
- 对于重要通知，建议先观察消息展示效果再扩大范围
